# 支援瀏覽器和語言版本

Next.js 支援 **IE11 以及其他現代的瀏覽器** (Edge, Firefox, Chrome, Safari, Opera 等) 且無需額外的配置。

## Polyfills

我們廣泛地注入 [polyfills](https://github.com/vercel/next.js/blob/canary/packages/next-polyfill-nomodule/src/index.js)，包含有 :

- [**fetch()**](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API) — 替換 : `whatwg-fetch` 和 `unfetch`.
- [**URL**](https://developer.mozilla.org/en-US/docs/Web/API/URL) — 替換 : [`url` package (Node.js API)](https://nodejs.org/api/url.html).
- [**Object.assign()**](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/assign) — 替換: `object-assign`, `object.assign` 和 `core-js/object/assign`.

如果你的相依性包含這些 polyfills，他們將會從生產版本中自動地被移除來避免相關地重覆。除此之外，為了減少 bundle 大小， Next.js 將只會為需要 polyfills 的瀏覽器載入相關資源。

### 客製 Polyfills

如果你的程式碼或是所需的任何外部 npm 相依性功能並不被目標瀏覽器支援 (例如： IE 11)，你需要自行加入 polyfills。

這個情況下，你需要加入一個最高層 import 以及所需的 **特殊 polyfill** 在你[客製 `<App>`](/docs/advanced-features/custom-app.md)或者是某個組件裡。

## JavaScript 語法特色

Next.js 允許你使用最新的 JavaScript，除了 [ES6 語法](https://github.com/lukehoban/es6features)， Next.js 也支援以下版本：

- [Async/await](https://github.com/tc39/ecmascript-asyncawait) (ES2017)
- [展開運算子 Object Rest/Spread Properties](https://github.com/tc39/proposal-object-rest-spread) (ES2018)
- [動態 `import()`](https://github.com/tc39/proposal-dynamic-import) (ES2020)
- [可選串連 Optional Chaining](https://github.com/tc39/proposal-optional-chaining) (ES2020)
- [空值合併運算子 Nullish Coalescing](https://github.com/tc39/proposal-nullish-coalescing) (ES2020)
- [Class Fields](https://github.com/tc39/proposal-class-fields) 和 [Static Properties](https://github.com/tc39/proposal-static-class-features) (階段 3 的提案一部分)
- 持續增加

### 伺服器端 Polyfills

除了在客服端的 `fetch()`，Next.js 的 polyfills `fetch()` 是在 Node.js 環境內。 你可以在伺服器端程式使用 `fetch()` (例如 `getStaticProps`/`getServerSideProps`) 而無需使用 polyfills，例如 `isomorphic-unfetch` 或 `node-fetch`。

### TypeScript 特色

Next.js 內建 TypeScript 支援，點擊[瞭解更多](/docs/basic-features/typescript.md)來看看怎麼使用。
